# /365 — Microsoft 365 Hub

## Instructions

When the user invokes `/365`, perform the following:

### 1. Connection Status Check

Check for M365 connectivity by verifying environment variables and token availability:

- Check if `AZURE_KEY_VAULT_URI` is set (vault connection for credentials)
- Check if `M365_CLIENT_ID` and `M365_TENANT_ID` are set (app registration)
- Check if a valid access token is cached or obtainable

Report status as one of:
- **Connected** — Token valid, Graph API reachable
- **Credentials Available** — Env vars set but no active token (may need auth flow)
- **Not Configured** — Missing environment variables

### 2. Display Available Sub-Commands

Present the following sub-commands:

```
Phoenix 365 — Microsoft 365 Integration
========================================

Connection: [status]

Available Commands:
  /mail              Email operations (list, read, send, search)
  /calendar          Calendar operations (today, week, create)
  /sharepoint        SharePoint operations (sites, lists, items, upload)

Quick Actions:
  /mail list         Show recent inbox messages
  /calendar today    Show today's schedule
  /sharepoint sites  List available SharePoint sites

Skills (auto-trigger):
  "check my email"          → reads inbox
  "what's on my calendar"   → shows schedule
  "find in SharePoint"      → searches SharePoint data
  "send an email to [name]" → composes and sends email
```

### 3. Quick Health Check

If connected, make a lightweight Graph API call to confirm the connection is live:
- `GET /me` — confirm the authenticated user identity
- Display: "Authenticated as: [user display name] ([email])"

### 4. Context for Phoenix Electric

This is a business tool for Phoenix Electric LLC, an electrical contracting company. Common use cases:
- Checking emails from customers and suppliers (electrical distributors, general contractors)
- Viewing the day's schedule (service calls, inspections, bids)
- Managing SharePoint lists (job tracking, customer database, inventory)
- Accessing OneDrive files (proposals, contracts, job photos, invoices)

Always present information in a business-practical format. Shane is the owner — he needs actionable summaries, not raw data dumps.
